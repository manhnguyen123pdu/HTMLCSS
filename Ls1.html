Kiến thức trọng tâm

INTERNET LÀ GÌ?

1.1 Khái niệm Internet

Định nghĩa: Internet (viết tắt của Interconnected Network) là một mạng lưới toàn cầu kết nối hàng triệu máy tính, thiết bị di động và hệ thống mạng nhỏ lại với nhau để chia sẻ thông tin.

Đặc điểm của Internet:

  • Toàn cầu: Phạm vi rộng khắp thế giới
  • Phân tán: Không do một tổ chức nào kiểm soát hoàn toàn
  • Giao tiếp đa chiều: Người dùng có thể gửi, nhận, tìm kiếm và lưu trữ thông tin
  • Mở: Dựa trên các giao thức mở, cho phép bất kỳ ai cũng có thể kết nối

Ví dụ về ứng dụng Internet:

Truy cập Google

Bạn dùng trình duyệt để truy cập Google.com ⇒ đang sử dụng Internet.

Gửi email

Gửi email qua Gmail ⇒ cũng là ứng dụng của Internet.

WEB LÀ GÌ?

1.2 Khái niệm Web (World Wide Web)

Định nghĩa: Web (World Wide Web - WWW) là một dịch vụ hoạt động trên nền Internet, cho phép người dùng truy cập và chia sẻ các trang web qua trình duyệt.

Lưu ý quan trọng

Internet là "hạ tầng", còn Web là "dịch vụ" chạy trên nền Internet. Nhiều người nhầm lẫn hai khái niệm này, nhưng chúng khác biệt về bản chất.

Các thành phần của Web:

Thành phần Giải thích ngắn gọn
Website Tập hợp nhiều trang web (web pages)
Trang web (Web page) Tài liệu HTML hiển thị qua trình duyệt
Trình duyệt (Browser) Công cụ truy cập web (Chrome, Firefox…)
URL Địa chỉ trang web (VD: https://google.com)
Địa chỉ IP Là 1 dãy số cung cấp nhận dạng cho 1 thiết bị sử dụng internet (VD: 192.168.45.15)
Tên miền (Domain name) Tên dễ nhớ đại diện cho địa chỉ IP (VD: google.com dễ nhớ hơn 142.250.66.14)
Web server Máy chủ lưu trữ và phục vụ các trang web

Ví dụ: Quy trình truy cập https://vnexpress.net

Bước 1: Gửi yêu cầu DNS

Trình duyệt gửi yêu cầu đến DNS server để tìm địa chỉ IP ứng với tên miền "vnexpress.net"

Bước 2: Tìm địa chỉ IP

DNS server tìm địa chỉ IP (ví dụ: 111.65.250.10) tương ứng với tên miền vnexpress.net

Bước 3: Gửi yêu cầu và nhận nội dung

Trình duyệt gửi yêu cầu đến IP đó ⇒ Nhận nội dung và hiển thị trang web

MÔ HÌNH CLIENT-SERVER

1.4 Cách trang web hoạt động (Mô hình Client – Server)

Quy trình hoạt động:

Bước 1: Người dùng nhập URL

Người dùng nhập URL vào trình duyệt (Client)

Bước 2: Trình duyệt gửi yêu cầu

Trình duyệt gửi yêu cầu đến Web Server

Bước 3: Server xử lý yêu cầu

Web Server xử lý và gửi lại trang web (HTML, CSS, JS…)

Bước 4: Hiển thị nội dung

Trình duyệt nhận và hiển thị nội dung cho người dùng xem

Minh họa mô hình:

Trình duyệt (Client)
Web Server
Gửi yêu cầu (Request)
Trả về nội dung (Response)

HTTP VÀ HTTPS

1.5 Giao thức HTTP / HTTPS

HTTP (Hypertext Transfer Protocol):

  • Là giao thức truyền dữ liệu giữa trình duyệt và web server
  • Dữ liệu không được mã hóa
  • Hoạt động trên cổng 80
  • Không an toàn, dễ bị tấn công nghe lén (sniffing)

HTTPS (HTTP Secure):

  • Là HTTP có bảo mật, dùng chứng chỉ SSL/TLS để mã hóa dữ liệu
  • Dữ liệu được mã hóa từ đầu đến cuối
  • Hoạt động trên cổng 443
  • An toàn hơn khi truy cập ngân hàng, thanh toán...
  • Có biểu tượng ổ khóa trên thanh địa chỉ trình duyệt
Tiêu chí HTTP HTTPS
Bảo mật Không an toàn An toàn
Cổng mặc định 80 443
Mã hóa Không Có (SSL/TLS)
Tốc độ Nhanh hơn Chậm hơn một chút
SEO Google ưu tiên thấp hơn Google ưu tiên cao hơn

CÔNG NGHỆ WEB CƠ BẢN

1.7 Các công nghệ phát triển web cơ bản

Công nghệ Vai trò chi tiết
HTML Là ngôn ngữ đánh dấu giúp xây dựng cấu trúc và nội dung cho trang web. HTML định nghĩa các thành phần như tiêu đề, đoạn văn, hình ảnh, liên kết… Đây là phần "bộ xương" của website.
CSS Dùng để trang trí và tạo kiểu dáng cho các thành phần HTML. CSS điều khiển màu sắc, font chữ, kích thước, bố cục, hiệu ứng chuyển động và làm cho website trở nên đẹp mắt, thân thiện.
JavaScript Là ngôn ngữ lập trình chạy trên trình duyệt, giúp tạo tính năng tương tác và xử lý sự kiện. Ví dụ: nút bấm, form nhập liệu, popup, hiệu ứng động, hoặc gọi API để lấy dữ liệu mà không cần tải lại trang.
PHP / Node.js Là các công nghệ phía server (máy chủ), chịu trách nhiệm xử lý logic nghiệp vụ, tương tác với cơ sở dữ liệu, quản lý phiên đăng nhập, gửi dữ liệu cho client.
MySQL / MongoDB Là các hệ quản trị cơ sở dữ liệu dùng để lưu trữ, truy vấn và quản lý dữ liệu của website như thông tin người dùng, bài viết, sản phẩm.
API Giao diện lập trình ứng dụng cho phép các ứng dụng khác nhau giao tiếp với nhau. RESTful API là phổ biến nhất trong phát triển web.
Framework Thư viện, công cụ hỗ trợ phát triển web nhanh hơn: React, Angular, Vue (front-end); Laravel, Express.js, Django (back-end).

Kiến trúc web hiện đại

Một website hiện đại thường có kiến trúc 3 lớp:

  1. Front-end: Giao diện người dùng (HTML, CSS, JavaScript)
  2. Back-end: Xử lý logic, kết nối database (PHP, Node.js, Python)
  3. Database: Lưu trữ dữ liệu (MySQL, MongoDB, PostgreSQL)
